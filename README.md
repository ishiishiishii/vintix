# 実験 README（ロボット歩行の汎化と高速適応）

## 目的（背景と課題）

ロボットの制御は、**ロボットの種類や環境条件が少し変わるだけで性能が落ち、学び直しが必要**になりがちです。  
既存の方法では、学習中に環境条件をランダムに変える（例: 地面の条件や初期状態を揺らす）ことで「特定条件への偏り」を抑えますが、**想定する多様性が増えるほど学習コスト（時間・計算）が増える**問題があります。

## 提案（本研究でやっていること）

この実験では、文章生成などにも使われる **Transformer** を用いて、ロボットの「過去の履歴」から次の行動を決める方式を扱います。

- **履歴の入力**: 状態・行動・報酬を時系列データとして入力し、次の行動を予測します。  
  ここでは「過去の成功・失敗の積み重ね」そのものを入力として使います。
- **課題意識**: 既存の履歴学習は「うまく制御できているデータ」中心になりやすく、ロボットや環境が変わると性能が大きく落ちることがあります。
- **アルゴリズム蒸留（学び方を学ぶ）**: 別の学習済みモデルが上達していく過程（最初は下手→失敗を重ねて改善）を教師データとして取り込み、**“上達のしかた” を学ぶ**ことを狙います。これにより、最初はうまく歩けなくても、履歴を手掛かりに徐々に行動を改善し、安定した制御に近づきやすくします。
- **モデル構造の工夫（共通部 + デコーダ）**:
  - **共通部**: ロボットに依存しにくい「一般的な上達のパターン」を学ぶ部分
  - **デコーダ**: 各ロボットに合わせて行動を出力する部分  
  こうすることで、未知ロボットに対しては **デコーダのみ追加学習**することで、モデル全体を作り直さずに適応することを狙います。

## 結果（要点）

学習データに含まれない **未知ロボット**に対し、「そのロボットを完全に制御できる専用モデル」と **平均累積報酬（エピソード中の報酬の合計）** を比較しました。  
その結果、**追加学習データを 10% に絞っても同等の結果**を得られました。

## 実験の流れ（このリポジトリで何をどこで実行するか）

このリポジトリでは、以下の流れで実験します（シミュレータは Genesis を利用）。

1. **専門家（うまく歩ける制御器）を学習**
   - 対象: Go1 / Go2 / A1 / MiniCheetah など
   - 実行: `Genesis/examples/locomotion/train.py`（PPO + MLP の専門家）
   - 何をしているか:
     - シミュレーション環境を立ち上げ、ロボットが歩くタスクを定義します。
     - 「観測（センサー情報など）」を入力に、「行動（各関節の指令）」を出す小さなニューラルネットを学習します。
     - 試行→評価→更新を繰り返し、累積報酬が高くなるように改善していきます。
     - 学習済みモデル（チェックポイント）を `Genesis/logs/...` に保存し、次の工程で利用します。
2. **擬似的な「上達の履歴データ」を作成**
   - 実行: `vintix_go2/scripts/collect_ad_data_parallel.py`
   - 何をしているか:
     - 上の「専門家」を使い、歩行の試行データを大量に生成します（並列実行で高速化）。
     - 1ステップごとに「状態・行動・報酬」を記録し、時系列の履歴データにします。
     - わざと条件を変えたり失敗を含めたりして、“上達の過程” っぽいデータ分布を作ります。
     - 生成した履歴データを `vintix_go2/data/` などに保存し、Transformer 学習の入力にします。
3. **Transformer（Vintix）を学習**
   - 実行: `vintix_go2/scripts/train_vintix.py`
   - 何をしているか:
     - 履歴データ（状態・行動・報酬の並び）を読み込み、学習用データセットを作ります。
     - Transformer が「過去の履歴」から「次の行動」を予測できるように学習します。
     - 複数ロボットを扱う場合は、共通部とロボット別の出力部（デコーダ）を分けて学習します。
     - 学習した重みを `vintix_go2/models/...` 等に保存し、評価スクリプトで読み込みます。
4. **評価（10 環境 × 10 エピソードで集計）**
   - 実行: `vintix_go2/scripts/eval_vintix.py`
   - 何をしているか:
     - 学習済み Vintix を Genesis の歩行環境に載せ、複数環境を同時に走らせます。
     - 既定で **10並列環境 × 各10エピソード**を実行し、累積報酬を集計します。
     - 結果を CSV やグラフとして保存し、ロボット間・条件間で比較できる形にします。
     - 必要に応じて動画保存もでき、挙動が直感的に確認できます。

## 実行環境（最短で動かす）

- Docker を使う場合は `CONTAINER.md` を参照してください（`docker-compose.yml` を同梱しています）。
- 実行時は基本的に、リポジトリのルート、または `vintix_go2/` 配下でスクリプトを動かします。
