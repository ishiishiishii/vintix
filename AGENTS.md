# AGENTS.md

四足歩行ロボの強化学習研究リポジトリ（Genesis + Vintix）。エージェントが守るべき共通ルールと、
詳細ドキュメントの所在をまとめる。実験固有の手順は、着手時に該当ドキュメント（後述）を読むこと。

## 1. プロジェクト概要
Transformer（Vintix）にロボットの「上達の履歴」を学習させ、未知ロボットへ
デコーダのみの追加学習で高速適応させる研究。シミュレータは Genesis。
専門家(PPO)の学習 → 履歴(AD)データ収集 → Transformer学習 → 評価、という流れで実験する。

## 2. 【最重要】実行ルール
- コードは **必ず Docker コンテナ内（`/workspace`）でのみ実行**する。ホスト側では実行しない。
- 現在のシェルがコンテナ内か不安なときは `cat /.dockerenv` と `whoami`(=kawa37) で確認する。
- 手動でのコンテナ運用が必要な場合のみ `CONTAINER.md` を参照（通常は DevContainer で自動接続）。

## 3. 【最重要】禁止事項（事故防止）
- 稼働中の実験プロセス（例: `run_experiment.py` / `collect_ad_data_parallel.py`）に**触れない**。
- プロセスの kill・再起動、`docker compose down/up`・再ビルドは**ユーザー確認なしに行わない**。
- 破壊的操作・スコープ変更は必ず事前確認する。
- 長時間ジョブは nohup でデタッチ起動されている場合がある。ログ更新が止まって見えても
  勝手に介入しない（別作業の影響のことがある）。

## 4. ディレクトリ構成
- `Genesis/examples/locomotion/` … シミュレータ環境と専門家(PPO)の学習・評価
  （`train.py` `eval.py` `*_env.py`、ロボット仕様は `robot_specifications_table.md`）
- `vintix_go2/scripts/` … 履歴収集・Transformer学習・評価の共通スクリプト群
- `vintix_go2/<実験名>/` … 個別実験の成果物と、その実験専用のコード・ドキュメント
- `vintix_go2/` 直下の `*.png` `*.csv` `logs/` `models/` `data/` `wandb/` … 生成物。基本いじらない。

## 5. 実験の進め方（ディレクトリ構成ルール）
- 新しい実験は、**内容を端的に表す名前**でディレクトリを作る（`vintix_go2/<実験名>/`）。
  **日付は付けない**。
- **共通・汎用のコードは `vintix_go2/scripts/`（や `Genesis/examples/locomotion/`）に置いて再利用する**
  （例: 専門家学習 `train.py`、Vintix学習 `train_vintix.py`、
  データ収集 `collect_ad_data_parallel.py`、評価 `save_vintix.py`）。
- **実験特有のコードはその実験ディレクトリに置く**。ただし**ゼロから実装せず、共通スクリプトの
  関数を import し、「データ収集 → 学習 → 評価」を呼び出すだけのオーケストレーション
  スクリプト**にする。ロジックを実験ディレクトリへコピー&改変するのは禁止
  （共通処理の修正が全実験に伝播しなくなるため）。
- 各実験ディレクトリには、目的・実行コマンド・パラメータなどの設定・結果サマリを書いた
  `README.md`（または `EXPERIMENT.md`）を置く。
- 既存実験に着手する際は、そのディレクトリの `README.md` / `EXPERIMENT.md` を先に読む。

## 6. 作業規約
- 生成物（`*.png` `*.csv` `*.pt` `*.log` `data/` `models/` `logs/` `wandb/`）は
  `.gitignore` 済み。基本的にコミットしない。
- API キー等の秘密情報はコードに書かず、環境変数（`.env`）で渡す。
- **作業の区切りごとに git でこまめにコミットする**。コミットメッセージは簡潔に「なぜ」を書く。
  ただし**稼働中の実験の生成物や別作業の変更は巻き込まない**（自分が変更・作成した
  ファイルだけを `git add` でステージする）。
- 作業の前に、関連ファイル・ドキュメントを能動的に参照（`@`参照や該当ファイルの読込）して
  文脈を確認してから進める。憶測でコードを書かない。

## 7. コミュニケーション規約
- ユーザーには**日本語**で対応する。
- 破壊的操作・スコープ変更は事前に合意してから進める。
